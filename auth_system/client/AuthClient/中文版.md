# AuthClient 编译与运行说明（中文版）

## 一、项目概述

本目录是 Win32 C++ 授权客户端 `AuthClient` 的源码目录。

当前默认构建特性如下：

- 目标架构：`x86`
- 编译工具链：`llvm-mingw` 中的 `clang++`
- 输出形式：单文件 `AuthClient.exe`
- 程序内置默认服务端地址：`http://159.75.226.54:8080`
- 启动权限：`requireAdministrator`

当前构建出来的 `AuthClient.exe` 以单文件运行为目标，不需要额外携带
`libc++.dll`、`libunwind.dll` 之类的 C++ 运行时 DLL。

## 二、主要文件说明

核心源码文件：

- `src/main.cpp`
- `src/auth_codec.h`
- `src/auth_codec.cpp`
- `build.bat`
- `AuthClient.exe.manifest`
- `AuthClient_manifest.rc`
- `auth_client.ini.example`

## 三、构建环境要求

建议环境：

- Windows
- 已安装 `llvm-mingw`
- `clang++` 已加入 `PATH`
- `windres` 已加入 `PATH`
- 可选：`llvm-strip` 已加入 `PATH`

当前 `build.bat` 默认使用：

```bat
set "LLVM_ROOT=E:\App\llvm-mingw-64"
```

如果你本机的 `llvm-mingw` 不在这个路径，请先修改 `build.bat` 中的：

```bat
set "LLVM_ROOT=..."
```

## 四、编译步骤

进入当前目录：

```bat
cd /d G:\code\c++\SuperSkillWnd\auth_system\client\AuthClient
```

执行编译：

```bat
build.bat
```

编译成功后会生成：

- 根目录输出：`AuthClient.exe`
- 打包目录输出：`dist\AuthClient\AuthClient.exe`

## 五、运行方式

可直接运行：

```bat
AuthClient.exe
```

也可以运行打包目录中的文件：

```bat
dist\AuthClient\AuthClient.exe
```

由于程序 manifest 已设置为 `requireAdministrator`，所以启动时会弹出 UAC
管理员授权提示。

## 六、默认服务端地址

当前程序会把默认服务端地址直接编译进程序：

```text
http://159.75.226.54:8080
```

这个值来自 `build.bat`：

```bat
set "DEFAULT_SERVER_URL=http://159.75.226.54:8080"
```

如果你想修改程序内置地址，只需要修改这一行后重新执行：

```bat
build.bat
```

## 七、可选外部配置文件

虽然当前程序已经内置了默认地址，但仍然支持通过同目录下的
`auth_client.ini` 覆盖默认配置。

示例配置：

```ini
server_url=http://159.75.226.54:8080
internal_server_url=
output_path=C:\Windows\hfy
timeout_ms=5000
```

如果 `AuthClient.exe` 同目录存在 `auth_client.ini`，则以该文件内容为准。

## 八、输出结果

正常编译完成后，通常会得到：

- `AuthClient.exe`
- `dist\AuthClient\AuthClient.exe`
- `dist\AuthClient\auth_client.ini.example`
- 如果本地存在 `auth_client.ini`，也会复制到 `dist\AuthClient`

## 九、常见问题

### 1. 提示 `clang++ was not found in PATH`

说明你的环境变量里没有 `llvm-mingw` 的 `bin` 目录。

可以先临时执行：

```bat
set PATH=E:\App\llvm-mingw-64\bin;%PATH%
```

然后重新运行 `build.bat`。

### 2. 提示找不到静态库

如果 `build.bat` 报缺少：

- `libc++.a`
- `libc++abi.a`
- `libunwind.a`

一般说明：

- `LLVM_ROOT` 路径写错了
- 或本机 `llvm-mingw` 安装不完整

### 3. 启动时弹管理员授权

这是正常行为，不是异常。

因为程序已经设置为：

- `requireAdministrator`

### 4. 当前服务器 IP 仍显示 `127.0.0.1`

这种情况通常说明：

- 程序内置的服务端地址仍指向本机
- 或当前目录下的 `auth_client.ini` 覆盖了内置配置

优先检查：

- `build.bat` 里的 `DEFAULT_SERVER_URL`
- 当前目录下是否存在 `auth_client.ini`

## 十、当前构建逻辑总结

当前 `build.bat` 主要完成这些工作：

- 编译 `x86` 版本
- 把 manifest 嵌入 `exe`
- 以单文件方式静态链接 C++ 运行时
- 输出可直接运行的 `AuthClient.exe`
- 同时生成 `dist\AuthClient` 目录

## 十一、建议使用方式

如果只是日常分发或部署，优先使用：

- `AuthClient.exe`

如果是交付给别人并顺带附带配置模板，使用：

- `dist\AuthClient`

如果是要交源码给别人重新编译，请使用：

- `build.bat`
- `BUILD_RUN.md`
- `中文版.md`
- `src` 目录
